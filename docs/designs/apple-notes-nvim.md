# CEO Plan: Apple Notes Neovim Plugin

Generated by /plan-ceo-review on 2026-03-23
Branch: main (greenfield) | Mode: SCOPE EXPANSION
Repo: apple-notes.nvim (local, no remote yet)

## Vision

### 10x Check

Not just "edit notes in neovim" — Apple Notes as a first-class neovim filesystem.
Notes organized in a tree view (neo-tree/oil.nvim), real-time search via Telescope,
quick capture without leaving your workflow, full folder management, trash access,
and checklist interactivity. A terminal developer forgets they're using Apple Notes.

### Platonic Ideal

A neovim user opens their editor and their notes are just... there. Folders in the
sidebar, fuzzy search across everything, `:w` syncs to their phone instantly.
Zero config — install the plugin, it discovers notes automatically. The experience
feels like editing local markdown files with magic sync. Conflict resolution when
the same note is edited on multiple devices simultaneously.

## Key Architecture Decisions

- **Approach:** Virtual Buffer (notes as in-memory buffers, no files on disk)
- **HTML↔MD:** pandoc (shell out). Swappable module for macOS 26+ native Markdown support.
- **SQLite Access:** Shell out to sqlite3 CLI (ships with macOS, zero dependencies)
- **Mutations:** AppleScript via osascript (only safe way to write to Apple Notes)
- **Async Model:** vim.fn.jobstart (callback-based, no external dependencies)
- **Schema Resilience:** Version-detection + adapter pattern for macOS version differences
- **Input Sanitization:** Dedicated sanitize.lua module for SQL + AppleScript escaping
- **External Change Detection:** Poll on BufEnter + 30s timer, prompt user on conflict

### Notification Design

- Use `vim.notify()` — falls back to `:echo` natively, styled by nvim-notify/fidget.nvim
- SUCCESS: INFO level, auto-dismiss 2s
- LOADING: WARN level, persistent (dismiss on completion)
- ERROR: ERROR level, persistent, always reference `:checkhealth apple-notes`

### New Note Behavior

- `:AppleNotesNew` → creates in default Notes folder (configurable)
- `:AppleNotesNew Work` → creates in "Work" folder directly
- Tree view: `a` on a folder → creates in that folder

### Delete Behavior

- Always confirms via `vim.fn.confirm("Delete '{title}'?", "&Yes\n&No", 2)` (default: No)
- Moves to trash (not permanent delete)
- Closes buffer if note was open

## Key Technical Notes

- Apple Notes stores content as HTML internally
- SQLite schema is Core Data-generated — column names may change between macOS versions
- AppleScript is the only safe way to mutate notes (direct SQLite writes corrupt Core Data)
- macOS 26 may support native Markdown in Notes — converter module designed as swappable
- Notes.app cold start takes up to 30s — all AppleScript calls must be async
- Password-protected notes cannot be accessed programmatically
- Database path: ~/Library/Group Containers/group.com.apple.notes/NoteStore.sqlite
- Full Disk Access (or terminal-level file access) required to read the database
